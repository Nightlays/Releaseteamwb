# GoalsPortal

React-микрофронт для портала `Построение целей команды`.

## Назначение

Модуль заменяет текущий процесс ведения целей в Google Docs и собирает в одном интерфейсе:

- квартал и год
- платформу / направление
- owner
- участников
- формулировку цели
- критерий успеха / KPI
- план исполнения
- статус и риск
- прогресс
- self-review и lead-review
- историю изменений

## Основные экраны

### 1. Обзор

Нужен для быстрого ответа на вопросы:

- сколько целей в квартале
- сколько активных и заблокированных
- где сейчас основные риски
- как выглядит поток целей по статусам
- как распределены цели по платформам

### 2. Доска

Kanban-представление для оперативного движения целей по статусам:

- Черновик
- Согласование
- В работе
- Квартальный review
- Закрыто
- Блокер

### 3. Реестр

Главный рабочий слой вместо Docs:

- все цели в одном плотном табличном виде
- фильтры по кварталу, платформе, owner, статусу
- быстрый переход к карточке
- экспорт JSON / CSV

### 4. Карточка цели

Полное рабочее место по одной цели:

- описание
- участники
- KPI
- план
- блокеры
- оценка
- история изменений

### 5. Люди

Срез по владельцам и участникам:

- сколько целей у owner
- сколько участий
- где критичный перегруз
- средний прогресс owner-целей

### 6. Настройки

MVP-справочники:

- платформы
- owner directory
- шкала оценки
- рабочий flow

## Рабочий флоу

1. Lead или owner создаёт цель.
2. Цель проходит согласование: уточняются owner, KPI, план, риски.
3. Во время квартала цель ведётся через статусы и обновления.
4. В конце квартала owner заполняет self-review.
5. Lead фиксирует lead-review и итоговую оценку.
6. Цель уходит в финальный квартальный результат.

## MVP-хранилище

Сейчас модуль работает на `localStorage` через `src/services/goalsPortal.ts`.

Это сознательно сделано как MVP-слой, чтобы:

- быстро интегрировать UI в портал
- не блокироваться на backend до проверки UX
- уже иметь доменную модель и экраны

## Что потребуется для полноценной интеграции

### Backend сущности

- `goal`
- `goal_participant`
- `goal_plan_item`
- `goal_update`
- `goal_evaluation`
- `goal_directory`

### API-контракты

- `GET /goals`
- `POST /goals`
- `PATCH /goals/:id`
- `POST /goals/:id/updates`
- `POST /goals/:id/review`
- `GET /goal-directories`
- `PATCH /goal-directories`

### Роли

- `admin`
- `lead`
- `owner`
- `participant`
- `viewer`

## Ограничения текущей версии

- нет backend persistence
- нет авторизации и ACL внутри самого модуля
- нет аудита на сервере
- нет workflow approval

Но UI, структура данных и пользовательский flow уже готовы под дальнейшую интеграцию.
