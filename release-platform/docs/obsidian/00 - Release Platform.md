# Release Platform

Release Platform - внутренний React/Vite портал для управления мобильными релизами WB: готовность Allure-прогонов, запуск релизов, SWAT/uWu, ЧП, BI-аудитория, графики, Wiki AI и доступы.

Открывать в Obsidian нужно папку `release-platform/docs/obsidian`.

## Что является текущим проектом

Документация описывает корневой проект `release-platform`.

В репозитории также есть `release_dashboard/release-platform`. Он похож на более старую или параллельную копию, но в корневом `release-platform` сейчас больше модулей и сервисов, поэтому он принят за активную версию.

## Техническая форма

- Frontend: React 18 + TypeScript + Vite.
- Навигация: общий shell в `src/App.tsx`, список модулей в `src/config/modules.tsx`.
- Микрофронты: папки `src/modules/*`.
- Сервисный слой: `src/services/*`.
- Shared UI: `src/components/ui`, `src/components/layout`, `src/components/charts`.
- Настройки: `SettingsContext`, `AppSettings`, `localStorage`.
- Legacy bridge: `legacy/*` копируется в `dist/legacy` через Vite plugin.

## Главный принцип архитектуры

Портал сейчас является frontend-heavy приложением: большинство доменной логики и интеграций живут в `src/services`, а backend заменяют локальный proxy, локальный LLM adapter, Google Apps Script, Supabase и внешние корпоративные API.
