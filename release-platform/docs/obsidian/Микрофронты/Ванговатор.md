# Ванговатор

## Назначение

Быстрый прогноз завершения Allure-прогонов по версии: загружает launches, считает total/finished/remaining/in progress и строит простой burndown/ETA.

## Где код

- UI: `src/modules/Vangovat/index.tsx`
- Реестр: `vangovat` в `src/config/modules.tsx`
- Legacy fallback: `legacy/Ванговатор.html`

## Данные и сервисы

- [[Allure TestOps]] через `src/services/allure.ts`:
  - `fetchLaunches`
  - `mapLaunch`
- Shared types: `AllureLaunchResult` из `src/types/index.ts`.

## Настройки

- `allureBase`
- `allureToken`
- `projectId`
- `proxyBase`, `proxyMode`, `useProxy`

## Что показывает

- прогнозное время завершения;
- скорость кейсов в час;
- процент пройденных кейсов;
- количество failed и in progress;
- список running launches с локальным ETA.

## Текущая точность

Скорость сейчас упрощена: `hoursSinceStart = 4` в UI. Для production-точности нужно брать реальные временные окна из Allure или переиспользовать модель [[Дашборд]].

## Что важно при рефакторинге

Модуль маленький и удобен как кандидат для выделения общего `AllureLaunchProgress` hook, который потом можно переиспользовать в [[Дашборд]] и [[Создание рана]].
