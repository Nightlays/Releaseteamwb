# Запуск релиза

## Назначение

Самый операционный модуль: автоматизирует запуск мажорного релиза и не-мажорных сценариев, создает/заполняет Allure runs, публикует сообщения в Band, добавляет дежурных, создает опросы, подтягивает компоненты и релизные данные.

## Где код

- UI: `src/modules/Launch/index.tsx`
- Основная логика: `src/services/launch.ts`
- Реестр: `launch` в `src/config/modules.tsx`
- Legacy fallback: `legacy/Запуск релиза.html`

## Данные и сервисы

- [[Allure TestOps]]:
  - создание runs;
  - поиск готовых runs;
  - заполнение major readiness run.
- [[Band]]:
  - feed post;
  - SWAT channel posts;
  - groups;
  - polls;
  - scheduled messages;
  - duty pings.
- [[Deploy Lab]]:
  - компоненты релиза.
- [[YouTrack]]:
  - релизные данные для major workflow.
- [[Google Apps Script и Drive]]:
  - duty editor tables и release notice data.
- [[Proxy]]:
  - маршрутизация browser requests.

## Основные режимы

- `major` - полный цикл Android + iOS.
- `hf_android` - хотфикс Android.
- `hf_ios` - хотфикс iOS.
- `napi` - Native API.
- `sunday_devices` - воскресные раны устройств.
- `rustore_critical` - RuStore/AppGallery Critical Path.
- `rustore_smoke` - RuStore/AppGallery Smoke.

## Настройки

- `allureToken`
- `bandCookies`
- `bandCookiesAdmin`
- `deployLabToken`
- `ytToken`
- `proxyBase`, `proxyMode`, `useProxy`

## Локальное состояние

- `wb_rl_step_schedules_v1`
- `wb_rl_automation_v1`
- `wb_release_launch_settings_v1`

## Риски

- Много write operations во внешние системы.
- Требуются admin cookies для части Band-действий.
- В одном service file смешаны Allure, Band, Deploy Lab, YT и текстовые генераторы.

## Что важно при рефакторинге

Не начинать с переписывания UI. Сначала разделить `launch.ts` на clients и workflow commands: `allureRunClient`, `bandClient`, `deployLabClient`, `majorWorkflow`, `nonMajorWorkflow`.
