# Создание рана

## Назначение

Инструмент создания Allure TestOps launches по готовым сценариям: ХФ Android/iOS, NAPI, воскресные устройства, RuStore/AppGallery Critical Path и Smoke.

## Где код

- UI: `src/modules/CreateRun/index.tsx`
- Сценарии: `src/services/createRun.ts`
- Дополнительная проверка launch list: `src/services/allure.ts`
- Реестр: `createrun` в `src/config/modules.tsx`
- Legacy fallback: `legacy/Создание рана.html`

## Данные и сервисы

- [[Allure TestOps]]:
  - создание tag релиза;
  - sync test plan;
  - запуск test plan;
  - чтение созданных launches.
- [[Proxy]] для обхода CORS.

## Сценарии

| Mode | Назначение | Test plans |
|---|---|---|
| `1` | ХФ Android | `1998` |
| `2` | ХФ iOS | `3258` |
| `3` | NAPI | `1036`, `1869`, `1871` |
| `4` | Воскресные устройства | набор device runs |
| `5` | RuStore/AppGallery Critical Path | storefront critical path |
| `6` | RuStore/AppGallery Smoke | `999`, `1000` |

## Настройки

- `allureBase`
- `allureToken`
- `projectId`
- `proxyBase`, `proxyMode`, `useProxy`

## Локальное состояние

- `create_run_react_state_v1` - выбранный релиз и режим.

## Что важно при рефакторинге

Это модуль с побочными эффектами во внешней системе. Нужны dry-run режим, idempotency checks и лог операций перед переносом логики в backend.
