# ЧП по стримам

## Назначение

Собирает количество принесенных ЧП по Android/iOS за диапазон релизов, раскладывает по стримам/сабстримам, сравнивает composition с GitLab MR и умеет выгружать результат в Google Drive.

## Где код

- UI: `src/modules/ChpCollect/index.tsx`
- Сервис: `src/services/chp.ts`
- Реестр: `chp` в `src/config/modules.tsx`
- Legacy fallback: `legacy/Сбор ЧП.html`

## Данные и сервисы

- [[Deploy Lab]]:
  - список issues по релизу и платформе;
  - cutoff/deploy data.
- [[YouTrack]]:
  - metadata issue, stream/substream, tags, release fields.
- [[GitLab]]:
  - merged MR composition по target branch.
- [[Google Apps Script и Drive]]:
  - upload итоговых таблиц.
- [[Proxy]]:
  - сетевой gateway.

## Настройки

- `deployLabToken`
- `ytToken`
- `gitlabToken`
- `proxyBase`, `proxyMode`, `useProxy`

## Локальное состояние

- `chp_turtles_v1_0_3`
- `chp_turtles_v1_0_2`

## Что показывает

- итоги по Android/iOS;
- стримовый обзор;
- таблицу по релизам;
- composition summary;
- логи сбора и upload.

## Что важно при рефакторинге

`chp.ts` является хорошим примером, где надо разделить внешние clients и чистую агрегацию. Особенно важно сохранить нормализацию release labels и stream/substream rules.
