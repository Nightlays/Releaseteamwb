# Цели команды

## Назначение

Портал целей команды: квартальные цели, owners, участники, KPI, план, статусы, риски, self-review, lead-review и история изменений.

## Где код

- UI: `src/modules/GoalsPortal/index.tsx`
- Доменная модель и persistence: `src/services/goalsPortal.ts`
- Локальная README: `src/modules/GoalsPortal/README.md`
- Реестр: `goals` в `src/config/modules.tsx`

## Данные и сервисы

- Сейчас backend отсутствует.
- Состояние хранится в `localStorage` через `goalsPortal.ts`.
- В сервисе есть доменные helpers:
  - `loadGoalsPortalState`
  - `saveGoalsPortalState`
  - `createEmptyGoal`
  - `createGoalUpdate`
  - `exportGoalsCsv`

## Основные экраны

- Обзор квартала.
- Kanban-доска.
- Реестр целей.
- Карточка цели.
- Люди.
- Настройки справочников.

## Роли внутри модуля

Модуль имеет собственную модель ролей Goals Portal, отдельную от shell RBAC: lead, owner, participant/viewer-like сценарии. Shell доступ все равно определяется через [[Управление доступом]].

## Ограничения

- Нет server persistence.
- Нет серверного аудита.
- Нет backend ACL.
- Нет настоящего workflow approval на сервере.

## Что важно при рефакторинге

Это хороший кандидат на первым backend contract: модель уже доменная и изолированная. Перед интеграцией стоит зафиксировать сущности `goal`, `goal_participant`, `goal_plan_item`, `goal_update`, `goal_evaluation`, `goal_directory`.
