# Анализ релизов за квартал

## Назначение

Квартальная таблица релизов и хотфиксов: версии, стримы, сабстримы, локомотивные задачи, даты сборки/отведения/отправки, раскатка и причины ХФ.

## Где код

- UI: `src/modules/ReleaseQuarterAnalysis/index.tsx`
- Сбор данных: `src/services/releasePages.ts`
- Persistence: `src/services/releaseQuarterSupabase.ts`
- Supabase schema: `supabase/release_quarter_analysis.sql`
- Реестр: `releaseAnalysis` в `src/config/modules.tsx`
- Legacy fallback: `legacy/Анализ релизов за квартал.html`

## Данные и сервисы

- [[Deploy Lab]]:
  - release summary;
  - issues;
  - deploy/cutoff data.
- [[YouTrack]]:
  - issue metadata, tags, stream/substream, hotfix fields.
- [[GitLab]]:
  - pipeline/job timings.
- [[Band]]:
  - milestones: build TF, entered review, planned send, rollout percent.
- [[Supabase]]:
  - tables `release_quarter_android`, `release_quarter_ios`.
- [[Proxy]]:
  - gateway для external API.

## Настройки

- `deployLabToken`
- `ytToken`
- `gitlabToken`, `gitlabCookie`
- `bandCookies`
- `proxyBase`, `proxyMode`, `useProxy`

## Что показывает

- таблицу по Android/iOS;
- фильтры по году, кварталу, месяцам, платформе, stream/substream;
- графики и срезы по локомотивам;
- сохранение/загрузку в Supabase;
- настраиваемую видимость/ширину колонок.

## Локальное состояние

- `rp_release_quarter_major_release`
- `rp_release_quarter_major_release_to`
- `rp_release_quarter_visible_columns`
- `rp_release_quarter_column_widths`

## Что важно при рефакторинге

Сохранить совместимость Supabase payload: `releaseQuarterSupabase.ts` хранит и плоские колонки, и `row_payload`. Перед изменением схемы нужен миграционный план.
