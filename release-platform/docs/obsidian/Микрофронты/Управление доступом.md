# Управление доступом

## Назначение

Админский экран управления пользователями, ролями и доступом к модулям. Работает поверх RBAC config и local overrides.

## Где код

- UI: `src/modules/AccessPanel/index.tsx`
- RBAC service: `src/services/legacy.ts`
- Config source: `legacy/rb_roles_access.json`
- Реестр: `access` в `src/config/modules.tsx`

## Данные и сервисы

- `legacy.ts`:
  - `loadRbacConfig`
  - `persistLocalUsers`
  - `persistLocalRoleAccess`
  - `persistLocalUserAccess`
  - `clearLocalAccessOverrides`
  - `canAccessModule`
  - `resolveAllowedLegacyIds`

## Роли shell

- `superadmin`
- `admin`
- `manager`
- `analyst`
- `viewer`

## Локальное состояние

- `rb_users_override_v1`
- `rb_role_access_v1`
- `rb_user_access_v1`
- auth keys: `rb_auth_ok_v1`, `rb_user_v1`, `rb_role_v1`

## Правила доступа

- `superadmin` и `admin` видят `access-management`.
- `superadminOnly` модуль доступен только `superadmin`.
- `explicitAccess` не попадает в default role access автоматически.
- `*` в role access означает все не-explicit модули плюс явно перечисленные.

## Риски

Это клиентский RBAC. Он контролирует видимость UI, но не защищает внешние API, если у пользователя есть токены/cookies.

## Что важно при рефакторинге

Если появится backend, этот экран должен стать UI над server-side ACL. До этого нельзя считать его security boundary.
