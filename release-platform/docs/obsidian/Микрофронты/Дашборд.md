# Дашборд

## Назначение

Главный экран готовности релиза: собирает состояние Allure-прогонов, считает общий прогресс, ручную активность, uWu-остаток, readiness по платформам и прогноз риска/ETA.

## Где код

- UI: `src/modules/Dashboard/index.tsx`
- Реестр: `dashboard` в `src/config/modules.tsx`
- Legacy fallback: `legacy/dashboard_run_module.html`

## Данные и сервисы

- [[Allure TestOps]] через `src/services/allure.ts`:
  - `fetchDashboardAggregate`
  - `fetchReadinessSummary`
  - dashboard cache helpers
- `src/services/releasePrediction.ts`:
  - `buildDashboardPrediction`
  - CatBoost ONNX risk model
  - operational fallback
  - people-aware forecast
- `src/services/regressionHistory.ts` используется из prediction layer для baseline.
- [[Google Apps Script и Drive]] используется через `GOOGLE_APPS_SCRIPT_URL` для исторических данных.

## Настройки

- `allureBase`
- `allureToken`
- `projectId`
- `proxyBase`, `proxyMode`, `useProxy`
- `mlHelperBase` косвенно влияет на доступность ML-потока.

## Что показывает

- readiness по Android/iOS;
- группы Allure: Critical, Smoke, Selective, Regression и связанные с ними счетчики;
- остаток, in progress, assigned activity;
- предупреждения по пустым или подозрительным запускам;
- прогноз дедлайна, риск и причины риска;
- историю снимков релиза.

## Локальное состояние

- `rp_dashboard_last_v1`
- `rp_dashboard_history_v1`
- кеши внутри `allure.ts`, включая release cache и readiness cache.

## Что важно при рефакторинге

Не смешивать три разных расчета: сырые Allure counts, readiness summary и predictive risk. Безопасная граница - сначала вынести pure model для `DashboardHistoryPoint` и `DashboardPredictionInput`, а UI оставить прежним.
