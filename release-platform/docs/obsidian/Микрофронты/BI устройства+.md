# BI устройства+

## Назначение

Отдельный микрофронт для набора готовых WB BI запросов по устройствам: производители, поиск устройств пользователя, доля платформ, tablets, iPad и FOLD.

## Где код

- UI: `src/modules/DeviceQueries/index.tsx`
- BI service: `src/services/youtrack.ts`
- Реестр: `deviceQueries` в `src/config/modules.tsx`
- Legacy fallback: нет, модуль React-only.

## BI queries

- `94951` - устройства по производителям.
- `94953` - устройства конкретного пользователя.
- `94954` - процент пользователей по платформам.
- `94955` - планшеты по `type=tablet`.
- `94956` - iPad по имени, более точный запрос.
- `94957` - FOLD по известному списку.
- `94958` - подсчет FOLD.
- `94959` - производители Android, от 100%.

## Поведение UI

- Переключатель запросов перенесен в верхний блок управления; кнопки компактные и без отображения query ID.
- Верхний поиск фильтрует текущую таблицу по всем колонкам.
- Для запроса `94953` доступно отдельное поле поиска пользователя; значение подставляется в подходящие `queryParams` сохраненного BI query и дополнительно применяется как локальный фильтр результата.
- Для каждого query строится аналитический блок по шаблону [[Популярные устройства]]: KPI, top-график, второй срез и предупреждения по автоопределению колонок.
- Для `94957` FOLD по списку добавляется нормализованная колонка модели и отдельная аналитика по распарсенным FOLD-моделям.
- Для запросов по производителям бренды нормализуются и объединяются: например `TECNO` + `TECNO MOBILE LIMITED`, `INFINIX` + `INFINIX MOBILITY LIMITED`; users/share суммируются.
- Таблица находится под графиками: sticky header, номер строки, горизонтальный скролл и ограничение видимых строк.
- Технические колонки `COL_1`, `COL_2`, `COL_3` заменяются на человекочитаемые заголовки по типу запроса.
- Графики живут в фиксированном canvas-контейнере, чтобы UI не растягивался вниз при перерисовке Chart.js.
- Есть последовательная загрузка всех запросов и CSV export текущего результата.

## Настройки

- `biCookie`
- `proxyBase`, `proxyMode`, `useProxy`

## Ограничения

Модуль запускает сохраненные BI query IDs так же, как страница WB BI `/env/all/queries/{id}`: сначала забирает описание из `/bi/v5/datasource/query/{id}`, затем отправляет body в `/bi/v2/queue/datasource/{queryId}`, опрашивает статус через connector dbconn и читает результат из cache-puller. Query ID остается `dataSourceId` в payload, как в WB BI.
