# Графики

## Назначение

Самый широкий аналитический микрофронт. Собирает метрики релизов за диапазон: test cases, coverage, selection, ЧП, типы задач, downtime, GitLab pipeline timings, rollout markers, anomalies, ML dataset и AI summary.

## Где код

- UI: `src/modules/Charts/index.tsx`
- Сервис: `src/services/charts.ts`
- Chart component: `src/components/charts/DonutChart.tsx`
- Реестр: `charts` в `src/config/modules.tsx`
- Legacy fallback: `legacy/Графики.html`

## Что показывает

- тренды по релизам;
- сравнение с предыдущим релизом или средним;
- coverage и selection;
- ЧП и типы ЧП;
- downtime/cut/store timing;
- stream delta insights;
- anomaly buckets;
- ML summary и LLM summary;
- PDF/export сценарии.

## Локальное состояние

- ML dataset keys `wb_graphs_v0_2_9_*`;
- AI summary cache в памяти;
- UI state для диапазона, сворачиваний, выбранных релизов.

## Риски

- `charts.ts` очень большой и совмещает clients, mapping, aggregation, ML и prompt building.
- Много частичных деградаций: один источник может упасть, но отчет должен остаться частично полезным.
- Есть CORS/cookie/token зависимости почти от всех внешних систем.

## Что важно при рефакторинге

Начинать с извлечения pure functions и typed DTO. Следующая безопасная граница - разделить источник данных по clients: `allureChartsClient`, `deployLabChartsClient`, `gitlabChartsClient`, `bandChartsClient`, `chartsMlService`, `chartsAiService`.
