# Пользователи по версиям

## Назначение

BI-отчет по пользователям мобильных версий: снимки аудитории iOS/Android, динамика между снимками, рост/падение, выборка top releases, PDF/CSV экспорт.

## Где код

- UI: `src/modules/BiUsers/index.tsx`
- BI service: `src/services/youtrack.ts`
- Proxy helper: `src/services/proxy.ts`
- Реестр: `biusers` в `src/config/modules.tsx`
- Legacy fallback: `legacy/BiUser.html`

## Данные и сервисы

- [[BI]]:
  - queue API;
  - cache-puller API;
  - datasource `11386`.
- [[Google Apps Script и Drive]]:
  - `bi_users_history.json`;
  - `wb_local_cache.json`.
- [[Proxy]]:
  - cookie forwarding через `X-Proxy-Cookie`.

## Настройки

- `biCookie`
- `proxyBase`, `proxyMode`, `useProxy`

## Локальное состояние

- `bi_users_calc_settings_v1`
- `bi_users_history`
- `wb_local_cache`

## Что показывает

- полный список релизов;
- top/selected view;
- дельты между снимками;
- тренды по iOS/Android;
- суммарные показатели по платформам.

## Риски

- BI работает через browser cookie.
- Есть локальный и Drive cache, поэтому важно понимать, какой snapshot считается актуальным.
- PDF export грузит внешние browser libraries runtime-скриптами.
