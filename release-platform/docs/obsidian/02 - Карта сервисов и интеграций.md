# Карта сервисов и интеграций

В проекте слово "сервис" в основном означает frontend service module из `src/services`. Отдельные локальные helper-сервисы лежат в `legacy` и `GLM`.

|---|---|---|

| `src/services/legacy.ts` | RBAC, auth в `localStorage`, ссылки на legacy assets, синхронизация старых ключей настроек. | Shell, [[Управление доступом]], legacy bridge |
| `src/services/allure.ts` | Allure launches, stats, member stats, readiness, dashboard aggregate, кеши. | [[Дашборд]], [[Ванговатор]], [[Создание рана]] |
| `src/services/releasePrediction.ts` | ETA/risk forecast, CatBoost ONNX, operational fallback, regression baseline. | [[Дашборд]] |
| `src/services/regressionHistory.ts` | История регрессионных прогонов через Google Apps Script и расчет baseline. | `releasePrediction.ts` |
| `src/services/createRun.ts` | Сценарии создания Allure runs и тегов для ХФ/NAPI/RuStore. | [[Создание рана]] |

## Внешние системы

| Система | Для чего используется | Настройки |
|---|---|---|
| [[Allure TestOps]] | Launches, test plans, readiness, member stats, test case fields. | `allureBase`, `allureToken`, `projectId` |
| [[YouTrack]] | Issues, Epic/User Story digest, release fields, ЧП metadata. | `ytBase`, `ytToken` |
| [[BI]] | Пользователи и устройства по версиям. | `biCookie` |
| [[Deploy Lab]] | Релизные issues, компоненты, статусы и даты релиза. | `deployLabToken` |
| [[GitLab]] | Pipeline timings, MR composition, build times. | `gitlabToken`, `gitlabCookie` |
| [[Band]] | Публикации релизов, SWAT/launch automation, rollout report. | `bandCookies`, `bandCookiesAdmin` |
| [[Google Apps Script и Drive]] | История, экспорты, отдельные JSON datasets. | встроенные Apps Script URL и Google auth на стороне script |
| [[Supabase]] | Персист квартального анализа. | publishable key в `releaseQuarterSupabase.ts` |
| [[GLM adapter]] | Локальный OpenAI-compatible proxy к CoreLLM. | `glmBase`, `glmKey`, `glmModel` |
| [[ML retrain helper]] | Локальный retrain CatBoost/ONNX. | `mlHelperBase` |
| [[Proxy]] | CORS-gateway для внешних API из браузера. | `proxyBase`, `proxyMode`, `useProxy` |

## Ключевая проблема для рефакторинга

Сервисный слой смешивает три ответственности: HTTP-клиенты, доменную агрегацию и UI-friendly formatting. Поэтому безопасный рефакторинг лучше начинать с выделения контрактов и adapter-слоя, не меняя экранов.
