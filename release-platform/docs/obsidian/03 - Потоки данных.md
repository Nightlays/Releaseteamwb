# Потоки данных

## Базовый runtime

1. `src/main.tsx` монтирует React SPA.
2. `SettingsProvider` читает `rp_settings_v1` и старые ключи legacy через `readLegacyBootstrapSettings()`.
3. `AppProvider` держит активный модуль и тему.
4. `Layout` грузит RBAC из `legacy/rb_roles_access.json`.
5. Пользователь логинится, shell фильтрует модули по роли и открывает `ModuleContent`.

## Внешний API-запрос

Типовой путь:

`Микрофронт -> useSettings -> src/services/<domain>.ts -> proxyFetch -> legacy/proxy-standalone.js -> внешний API`

Proxy нужен, потому что браузерные запросы к Allure, YouTrack, Band, GitLab, Deploy Lab и BI часто упираются в CORS или cookie/header constraints.

## Настройки и токены

Главный объект: `AppSettings` в `src/types/index.ts`.

Сейчас настройки живут в браузере:

- `rp_settings_v1` - новый общий формат.
- Старые ключи legacy - поддерживаются через `src/services/legacy.ts`.
- Модульные настройки - часто отдельные ключи `localStorage`, например `bi_users_calc_settings_v1`, `swat_release_react_v2`, `band_android_rollout_v1_*`.

## Legacy assets

`vite.config.ts` содержит `legacy-assets-bridge`:

- в dev server раздает файлы из `legacy/*` по `/legacy/<file>`;
- на build копирует `.html`, `.css`, `.js`, `.json`, `.mjs`, `.png`, `.svg`, `.onnx` в `dist/legacy`;
- поддерживает Unicode alias для имен файлов.

## Кеши и persistence

| Тип данных | Где хранится |
|---|---|
| Настройки платформы | `localStorage: rp_settings_v1` |
| Auth/RBAC overrides | `localStorage: rb_*` |
| Dashboard readiness/cache | `localStorage`, ключи внутри `allure.ts` |
| Goals portal | `localStorage`, `goalsPortal.ts` |
| BI history/cache | `localStorage` + Google Apps Script/Drive |
| Charts ML dataset | `localStorage` + Google Drive Apps Script |
| Quarter analysis | Supabase tables `release_quarter_android`, `release_quarter_ios` |
| Legacy HTML/modules | `legacy/*`, `dist/legacy/*` |

## Самые плотные интеграционные цепочки

- [[Дашборд]]: Allure aggregate -> local cache -> `releasePrediction.ts` -> CatBoost ONNX или operational fallback -> UI.
- [[Графики]]: release range -> Allure/YT/DeployLab/GitLab/Band/Wiki -> ML dataset -> AI summary -> charts/PDF/export.
- [[Запуск релиза]]: release + mode -> Allure runs -> Band posts/groups/polls -> Deploy Lab components -> YouTrack data.
- [[Анализ релизов за квартал]]: release range -> `releasePages.ts` external enrichment -> Supabase save/load.
- [[Wiki Intelligence v4]]: question -> Wiki search/web search -> GLM adapter -> answer/draft -> optional wiki publish.
